# Арт-Калькулятор v2.0 — Техническая документация

Учебно-тренировочный артиллерийский калькулятор для Arma 3 (RHS + ACE).
PWA, работает полностью офлайн после первой загрузки.

## Структура проекта

```
Project-1/
├── index.html              — UI (SPA)
├── app.js                  — бизнес-логика расчётов
├── style.css               — стили (glassmorphism, responsive)
├── sw.js                   — Service Worker (offline, версионный кэш)
├── manifest.json           — PWA-манифест
├── icon-192.png / icon-512.png
├── tables/
│   ├── tables.js           — автогенерируемый бандл данных
│   ├── update_tables.ps1   — генератор tables.js из CSV
│   ├── update_tables.bat   — обёртка для PS1
│   └── *.csv               — таблицы стрельбы (23 файла)
├── tests/
│   └── test_calculations.html — unit-тесты (открыть в браузере)
└── docs/
    └── TECHNICAL_DOCUMENTATION.md
```

#### `WEAPON_NAMES` — маппинг имён файлов → человекочитаемые названия
Содержит названия для всех 21 орудия/системы. При добавлении нового CSV
достаточно добавить запись в этот объект — если запись отсутствует,
отображается имя файла.

#### Функции

**`lerp(a, b, t)`** — линейная интерполяция: `(1-t)*a + t*b`

**`getVal(el, def)`** — числовое значение из input-элемента с fallback

**`calculateDensityPct(tempC, pressureHPa, humidity)`**
Расчёт отклонения плотности воздуха от ISA. Формула Тетенса для давления
насыщенного пара. Возвращает % отклонения.

**`parseCSV(text)`** — парсинг CSV (разделители: `,`, `\t`, `;`), пропуск заголовка

**`getWeaponDisplayName(key)`** — возвращает человекочитаемое имя из `WEAPON_NAMES` или ключ

**`findBracket(table, dist)`** — поиск двух строк таблицы для интерполяции.
Возвращает `{ row1, row2, factor }` или `null`.

**`loadDefaultTables()`** — загрузка из `tablesRawData` (tables.js), заполнение `<select>`

**`saveInputs()` / `restoreInputs()`** — сохранение/восстановление ввода через localStorage

**`normalizeAz(az)`** — нормализация азимута в диапазон [0, 6400)

**`calculate()`** — основной алгоритм расчёта (см. ниже)

**`displayResults(...)`** — отображение результатов и таблицы всех зарядов

## Алгоритм расчёта

1. **Геодезия**: `dist = √(dx² + dy²)`, `az = atan2(dx, dy)` → мил
2. **Ветер**: разложение на продольную/поперечную составляющие
   - `windTail = -V × cos(windDir - targetAz)` — попутный (>0) / встречный (<0)
   - `windCross = -V × sin(windDir - targetAz)` — слева (>0) / справа (<0)
3. **Плотность воздуха**: отклонение от ISA через формулу Тетенса
4. **Для каждого заряда**:
   - Интерполяция метеокоэффициентов при геодезической дистанции
   - Расчёт поправки: `corrM = wind_corr + temp_corr + dens_corr`
   - Эффективная дистанция: `effDist = distGeo + corrM`
   - Интерполяция баллистики при эффективной дистанции
   - Коррекция на разность высот
   - Поправка азимута от бокового ветра
5. **Выбор**: заряд с минимальным TOF (+ таблица всех зарядов)

### Логика знаков метеопоправок

Знаки в CSV-колонках W_HEAD и W_TAIL установлены так, что при умножении
на абсолютную скорость ветра результат можно напрямую прибавить к дистанции:

- **Встречный ветер** тормозит снаряд → нужно стрелять дальше → `W_HEAD > 0`
- **Попутный ветер** ускоряет снаряд → нужно стрелять ближе → `W_TAIL < 0`

Аналогично для температуры и плотности.

## Добавление нового орудия

1. Создать CSV-файл в `tables/` с 13 колонками (формат см. выше)
2. Запустить `update_tables.bat` (или `.ps1`) — tables.js обновится автоматически
3. (Опционально) Добавить запись в `WEAPON_NAMES` в `app.js` для человекочитаемого имени
4. Увеличить `CACHE_VERSION` в `sw.js` для обновления кэша у пользователей

Если запись в `WEAPON_NAMES` не добавлена, в выпадающем списке отобразится
имя файла (BaseName CSV) — приложение продолжит работать корректно.

## Service Worker

Стратегия: Network First → обновление кэша → fallback на кэш при офлайне.
- `CACHE_VERSION` — инкрементировать при деплое
- `self.skipWaiting()` + `self.clients.claim()` — мгновенная активация нового SW
- Старые кэши удаляются автоматически при активации
